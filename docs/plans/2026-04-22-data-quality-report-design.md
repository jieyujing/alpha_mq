# 数据质量报告模块设计

## 背景

当前 `CSI1000QlibPipeline` 完成后无数据质量报告输出。用户无法直观了解：
- 数据覆盖情况（标的数、时间范围）
- 数据完整性（缺失值、重复行）
- 各类数据的一致性

## 目标

创建独立的数据质量报告模块，在 pipeline 完成后自动生成 Markdown 格式的质量报告。

## 设计决策

### 输出时机

**所有 stages 完成后汇总输出** — 在 `teardown()` 中调用报告生成器。

### 报告内容

**详细质量指标** — 基础统计 + 数据完整性检查（空值、异常值、重复行）+ 一致性校验。

### 输出格式

**Markdown 文件** — 保存为 `data/qlib_output/quality_report.md`，便于阅读和归档。

### 架构方案

**独立 QualityReporter 模块** — 创建 `src/pipelines/data_quality/` 目录，模块化设计便于复用。

---

## 模块结构

```
src/pipelines/data_quality/
├── __init__.py              # 导出 QualityReporter
├── reporter.py              # QualityReporter 类（核心）
└── checks.py                # 各类检查函数
```

**职责划分**：
- `checks.py`: 单项检查函数（`check_ohlcv_coverage`, `check_missing_values`, `check_duplicates` 等）
- `reporter.py`: 组合检查结果，生成 Markdown 报告，保存文件
- Pipeline 在 `teardown()` 中调用 `QualityReporter`

## QualityReporter 类

```python
class QualityReporter:
    """数据质量报告生成器"""
    
    def __init__(self, config: dict):
        self.exports_base = Path(config.get("exports_base", "data/exports"))
        self.qlib_output = Path(config.get("qlib_output", "data/qlib_output"))
        self.qlib_bin = Path(config.get("qlib_bin", "data/qlib_bin"))
        self.report_path = self.qlib_output / "quality_report.md"
        
    def run_all_checks(self) -> dict:
        """执行所有检查，返回结果字典"""
        
    def generate_markdown(self, results: dict) -> str:
        """将检查结果转为 Markdown 格式"""
        
    def save_report(self) -> Path:
        """执行检查并保存报告，返回报告路径"""
        
    # 内部检查方法
    def _check_ohlcv(self) -> dict:
        """检查 OHLCV 数据"""
        
    def _check_features(self) -> dict:
        """检查估值/市值数据"""
        
    def _check_pit(self) -> dict:
        """检查 PIT 数据"""
        
    def _generate_summary(self) -> dict:
        """汇总统计"""
```

---

## 检查项详情

| 检查类别 | 检查项 | 说明 |
|----------|--------|------|
| **OHLCV** | 标的数量 | 统计 history_1d 目录下的文件数 |
| | 时间覆盖 | 每个标的的起止日期，计算覆盖率 |
| | 缺失值 | 检查 open/high/low/close/volume 空值占比 |
| | 重复行 | 检查 (symbol, date) 重复的行数 |
| | 异常值 | 检查 close <= 0 或 volume < 0 的行数 |
| **Features** | 标的覆盖 | 对比 OHLCV 标的，检查覆盖率 |
| | 缺失值 | 检查 pe_ttm, pb_mrq 等空值占比 |
| **PIT** | 标的覆盖 | 检查财务数据覆盖的标的数 |
| | 报告期覆盖 | 检查财报披露日期的时间范围 |
| **Summary** | 总文件数/大小 | 统计目录文件数和磁盘占用 |
| | 整体评分 | 综合覆盖率，给出整体评分 |

## Pipeline 集成

```python
# csi1000_pipeline.py 的 teardown() 中

def teardown(self):
    from data_quality import QualityReporter
    reporter = QualityReporter(self.config)
    report_path = reporter.save_report()
    logging.info(f"Quality report saved: {report_path}")
```
